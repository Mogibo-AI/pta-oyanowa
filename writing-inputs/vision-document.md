## Executive Summary (エグゼクティブサマリー)

全自動PTA - おやのわ は、PTA活動をAIが代行することで「誰も役員をやらなくていい、全員が参加している組織」を実現するSaaSアプリです。
保護者はアプリをインストールして気が向いたときに意見を投稿するだけで、AIが集約・企画・調整・記録・報告のすべてを行います。
既存のPTA管理サービスが「役員の業務を効率化する」だけであるのに対し、本サービスは「役員という役割そのものを不要にする」ことを目指します。
学校単位のSaaSモデルで提供し、保護者は無料で利用でき、学校（または自治体）が月額費用を負担します。
PTAの廃止ではなく、PTAを「誰もが無理なく続けられる仕組み」に変えることが本質的なゴールです。

## Problem Statement (問題ステートメント)

### 既存サービスの限界

現在市場に存在するPTA関連サービスは、**PTA役員を1人立てて管理しやすくするもの**がほとんどです。
これでは役員の業務が多少効率化されるだけで、以下の根本課題は何も解決されていません：

- **「誰かが役員にならなければならない」という強制構造**
- 学校行事の企画・保護者間の連絡調整・会費管理という重い責任
- 役員になりたくないという心理的負担とそれに伴う人間関係の摩擦

### PTAが抱える構造的欠陥

| 問題 | 実態 |
|---|---|
| 役割の強制輪番 | やりたくない役が年に一度必ず回ってくる。断ると空気が悪くなる |
| 参加の不平等 | 声の大きい保護者・役員経験者の意見が通りやすい |
| 時間の強制拘束 | 平日昼間の会議。働く親は参加できない |
| 形骸化した民主主義 | 「意見を聞く」→ 実質すでに役員が決めている |
| 引き継ぎの属人化 | 担当者が変わるたびにノウハウが消える |

### 結果として起きていること

- 役員のなり手がいない → 毎年くじ引き・押しつけ合い
- 参加率が下がる → 一部の人に負担が集中
- 不満はあるが言えない → 組織の改善が起きない
- PTAをやめたくても慣習で続けざるを得ない

### 解決アプローチ

PTAを廃止するのではなく、**「PTAから人間がやるべき仕事を限界まで除去する」**。
AIが企画・調整・連絡・記録・報告を担い、人間の判断は「賛否投票」と「タスクの受諾/辞退」のみとする。

## Target Users (ターゲットユーザー)

| ユーザー種別 | 主な特徴 | このサービスで解決されること |
|------------|---------|--------------------------|
| **保護者（一般）** | 共働き・時間がない・役員はやりたくない | 会議に出なくていい。アプリで意見を投稿するだけで参加できる |
| **保護者（意欲的）** | イベントを企画したい・意見を言いたい | 目安箱・アンケートで意見が可視化され、支持されれば実現する |
| **学校管理者（校長・教頭）** | PTAとの調整が手間・何を依頼すればいいか不明 | 管理画面で全体を把握。目安箱の意見が自動メール通知で届く |
| **学校の先生（担当）** | PTAに関する保護者意見の窓口役を担う | 管理画面からメール自動送信 or 手動確認を選択できる |
| **PTA現役役員** | 来年はやりたくない・引き継ぎが大変 | 役員としての承認操作は不要。学校側の管理者担当者が管理画面で対応。引き継ぎは自動 |
| **自治体担当者** | 管轄内の複数校でPTA問題を抱えている | 自治体一括プランで複数校を一元管理できる |
| **新入学保護者** | PTAの慣習を知らない。最初から新しい仕組みで参加したい | アプリをインストール→ログイン→学校コード入力でオンボーディング完了 |

## Full Scope Vision (フルスコープビジョン)

成熟した「全自動PTA - おやのわ」が実現するすべての機能。

### 機能エリア 1: デジタル目安箱

- テキスト・画像投稿対応
- 匿名/実名選択
- AIによるカテゴリ分類（安全・イベント・連絡・要望・その他）
- 類似投稿の自動グルーピングと「支持数」可視化
- 対応状況（未対応/検討中/実施済み）の公開管理
- **ポジティブな提案（イベント希望）も同等に扱う**

### 機能エリア 2: AI アンケート自動生成

- 学校行事・季節・過去意見傾向を踏まえた質問自動生成（5〜7問）
- プッシュ通知での全員配信
- リアルタイムグラフ表示
- 未回答者への自動リマインド
- 「あなたの意見が反映されました」という参加感の可視化

### 機能エリア 3: タスク自動割り当て・進捗管理

- イベント確定後のタスク自動洗い出し
- 参加実績・希望時間帯・タスクの重さを考慮した自動割り当て
- 連続タスク集中の自動分散
- 引き継ぎ資料の自動生成
- **前日AIマネジメント機能**（前日のみ解放される専用ページ・リアルタイム指示・サポートAI）

### 機能エリア 4: イベント企画 AI

- 目安箱・アンケート・学校カレンダーを統合した自動企画生成
- 実施候補日・必要人手・想定参加者数の自動算出
- 全保護者への賛否投票
- 賛成多数なら実施フローに自動移行

### 機能エリア 5: 透明な月次レポート

- 活動内容・予算・決定事項のリアルタイム公開
- 月次レポートの自動生成・全員配信

### 機能エリア 6: 過去情報・履歴ダッシュボード

- 過去イベント一覧・評価・参加人数
- 会費収支グラフ（月別・年別）
- 参加率推移・アンケート回答率推移
- 目安箱投稿数・カテゴリ別推移

### 機能エリア 7: カレンダー表示

- 学校行事カレンダーとAI提案イベントカレンダーの統合表示
- 確定/未確定（?マーク）の視覚的区別
- カレンダーから賛否投票・イベント詳細・アンケートへの遷移

### 機能エリア 8: 会費徴収・管理

- 学校管理者による年間会費・緊急追加会費の設定
- AIによる会費改善提案（予算消化率から最適額を算出）
- 保護者向け会費明細・領収書表示
- AIは会費を自動変更しない（必ず管理者操作が必要）

### 機能エリア 9: 学校管理者画面

- 管理者専用ログイン（二要素認証）
- 破壊的操作時の「保護者が見れる操作ログに残ります」警告ダイアログ
- 表示/非表示設定（学校が制御）
- 非表示の操作ログは項目名のみ保護者に開示
- AIは管理者設定の変更不可

### 機能エリア 10: 議事録・会議サポート

- 目安箱の「集まって議論したい」意見を検知してAIが会議開催を提案
- 管理者が任意でいつでも会議を開催可能
- カレンダーから「議事録・録音開始」ボタンをタップするだけで開始
- 突発的な議論にも対応（カレンダー登録なしでも即開始可）
- AI がリアルタイムで音声文字起こし・話者識別・要点抽出
- 会議終了後に議事録を自動フォーマット化して全保護者に公開
- 決定事項をアプリのアクションリストに自動登録

### 機能エリア 11: アンケート意見主張トグル（先生へのエスカレーション）

- 各アンケート回答に「意見を主張する」トグルを設置
- トグルOFF（通常）: AIが意見を集約・対応策を提案し保護者全体で議論
- トグルON（主張モード）: 学校の先生に直接エスカレーション
  - 対象意見が先生にも見える状態になる
  - 先生に意見内容のメールが届く（管理者が自動送信 or 手動確認を選択可）
  - 「先生が確認中」ラベルが保護者にも表示される
- エスカレーション先（担任/教頭/校長）は管理者が設定可能

### 機能エリア 12: 自治体・複数校管理

- 市区町村単位での複数校一括管理
- 校間での活動事例共有（匿名化）
- 自治体向けダッシュボード（集計レポート）

| プッシュ通知 | アンケート配信通知・月次レポート配信通知 | 回答率を上げる必須機能 |

## Risks and Open Questions (リスクとオープンクエスチョン)

### リスク

| リスク | 影響度 | 対策 |
|-------|-------|------|
| 個人情報保護法・学校情報の扱い | 高 | 学校単位でデータ分離 / 児童個人情報はシステムに入れない / ISMS取得を目指す |
| 学校側の承認・意思決定が遅い | 中 | 自動承認モードを用意。管理画面で切り替え可能にする |
| 保護者のアプリインストール率が低い | 高 | 入学式・懇談会でのQRコード配布からアプリインストール→ログイン→学校コード入力の導線を整備 / Web版でもアクセス可能にする（Phase 2） |
| AIの企画・分類精度が低い | 中 | Claude Sonnetを使用 / 精度モニタリングを実装 / 全保護者の賛否投票と学校管理者の確認フローを維持 |
| 会費関連のトラブル | 高 | AIによる自動変更を不可能にする / 変更操作ログを保護者に公開する |
| 学校IT担当者のリテラシー不足 | 中 | オンボーディングサポート（リモート）を導入時に提供 |
| 競合サービスの参入 | 低〜中 | 操作ログ公開・AI設定変更不可という「透明性設計」で差別化 |
| AIの不適切な発言・提案 | 高 | 学校・保護者に関わる文脈でのガードレール設定 / 全提案は保護者投票で承認必須 |
| 音声録音・文字起こしの精度（議事録機能） | 中 | 複数人・雑音環境での認識精度を事前検証 / 手動修正機能を必ず用意する |
| 意見主張トグルの誤用（先生への不適切なエスカレーション） | 中 | AI によるフィルタリング（明らかに不適切な内容は送信前に警告） / 送信確認ダイアログを表示 |

## Competitive Analysis (競合分析)

| サービス種別 | 主な機能 | 全自動PTAとの差別化 |
|------------|---------|------------------|
| PTA管理SaaS（既存） | 役員1名が使う管理ツール。出席管理・配布物デジタル化 | 「役員が必要」という前提を変えない。本サービスは役員自体を不要にする |
| 学校連絡アプリ（マチコミ等） | 学校→保護者への一方的な連絡配信 | 双方向の意見収集・AI企画生成・タスク割り当てがない |
| 汎用アンケートツール（Forms等） | アンケート作成・収集 | AI自動生成なし。学校行事との連携なし。PTA専用設計でない |
| SNSグループ（LINE等） | 非公式の連絡手段 | 全員に届かない。記録が残らない。AI機能なし |

**差別化の核心**: 「透明性」と「AI自動化」の組み合わせ。
会費変更・提案削除などの操作ログが保護者に公開されること、AIが設定を勝手に変更できないことで、**学校への信頼と保護者の安心感を担保**します。

## Appendix: User Flow (付録: ユーザーフロー全体)

```
① 学校がアプリを導入（管理者アカウント設定）
      ↓
② 保護者にQRコードを配布 → アプリインストール → ログイン → 学校コードを入力
      ↓
③ 保護者は気になったときに目安箱に投稿
      ↓
④ AI が月1回アンケートを全員に配信（プッシュ通知）
      ↓
⑤ AI が投稿・回答を集約してイベント案を自動生成
      ↓
⑥ 全保護者が賛否投票（1タップ）
      ↓
⑦ 可決されたらタスク割り当て・担当者に個別通知
      ↓
⑧ 実施 → AI が結果をレポートとしてまとめ全員に配信
      ↓
⑨ 翌年の担当者に引き継ぎ資料が自動生成される
```

人間が判断する場面は「賛否投票」と「タスクの受諾/辞退」だけ。
