## P-01: 田中ゆかり（一般保護者）

### 基本情報
| 項目 | 値 |
|---|---|
| 年齢 | 38歳 |
| 職業 | 会社員（フルタイム共働き） |
| 子ども | 小学3年生（1人） |
| PCリテラシー | 中（スマホ中心、PCは必要時のみ） |
| 主要デバイス | iPhone（iOS最新） |
| PTA歴 | 役員経験なし |

### モチベーション
- 子どもの学校に関心はあるが、平日昼間の会議には参加できない
- 自分の意見が学校運営に反映されるならアプリでの参加は歓迎
- 「役員にされる」恐怖から PTA から距離を置きたい

### 困りごと
- 平日昼間の PTA 会議に参加できない（仕事中）
- LINE グループでの非公式連絡が雑多で重要情報が埋もれる
- 「誰かが役員にならなければ」という空気が重い

### このプロダクトで解決されること
- 会議に出なくても、アプリで気になったときに意見を投稿できる
- アンケートに1〜2タップで回答するだけで参加感が得られる
- 役員という役割が存在しないため、押し付け合いが起きない

### 典型的な利用シナリオ（1ヶ月）
- 月初: 月次レポート（前月の活動）をアプリで確認
- 不定期: 子どもから聞いた学校の話題で気になることを目安箱に投稿（匿名選択）
- AI判断による配信日: プッシュ通知でアンケートが届き、通勤電車で1分で回答
- 不定期: 他保護者の意見に「支持」をタップ
- 不定期: イベント企画案に賛否投票（賛成 / 反対 / 興味なし）

### マッピング機能
F-01（投稿・閲覧・支持）/ F-02（アンケート回答）/ F-03（通知受信）/ F-04（レポート閲覧）/ F-05（ログイン）/ F-08（賛否投票）/ F-09（Coming Soon の将来期待感）

## P-02: 山本健太（意欲的保護者）

### 基本情報
| 項目 | 値 |
|---|---|
| 年齢 | 42歳 |
| 職業 | 自営業（フリーランスエンジニア） |
| 子ども | 小学5年生（1人）+ 小学2年生（1人） |
| PCリテラシー | 高（IT職種） |
| 主要デバイス | iPhone + Mac（PC利用も多い） |
| PTA歴 | 過去に1度役員経験あり、不満を持っている |

### モチベーション
- 学校行事の企画に積極的に意見を出したい
- 既存PTAの非効率さを変えたい
- 「やりたい人がやれる」フェアな運営を望んでいる

### 困りごと
- 既存PTAは「声が大きい人」が決める傾向で公平性に欠ける
- 自分の意見が反映されたか分かりにくい
- 提案しても「じゃあ役員やってください」と返される

### このプロダクトで解決されること
- 目安箱で具体的な提案を投稿でき、AI が他保護者の関心とマッチングして可視化してくれる
- 提案がイベント企画AIで形になり、全員投票で実現の可能性が見える
- 役員にならなくても提案できる仕組み

### 典型的な利用シナリオ（1ヶ月）
- 週1〜2回: 目安箱に提案を投稿（実名で）
- 不定期: 他人の投稿で気になるものに「支持」、コメント代わりの追加投稿
- アンケート配信日: 詳細回答 + 自由記述欄も活用
- イベント企画案表示時: 内容を読み込み、賛成/反対の判断後に投票
- 月初: 月次レポートで自分の意見の影響を確認

### マッピング機能
F-01（実名投稿・支持・閲覧）/ F-02（詳細回答）/ F-04（影響の確認）/ F-05 / F-08（企画案レビュー・投票）/ F-09

## P-03: 鈴木一郎（学校管理者・校長）

### 基本情報
| 項目 | 値 |
|---|---|
| 年齢 | 55歳 |
| 職業 | 公立小学校 校長 |
| PCリテラシー | 中（メール・Excel は使える、新しいツール導入には慎重） |
| 主要デバイス | 学校PC + 個人iPhone |
| PTAとの関わり | 学校行事の調整・PTA総会出席 |

### モチベーション
- PTA 役員のなり手不足を本気で解決したい
- 学校と保護者の信頼関係を維持したい
- 操作ログ公開などの「透明性」を担保したい
- 個人情報・会費の不適切操作で批判されるのは絶対に避けたい

### 困りごと
- PTA 役員選出で毎年「押し付け合い」が発生し空気が悪くなる
- 保護者からの意見集約に時間がかかる
- PTA 関連の業務が校務を圧迫
- 既存ITサービス導入には学校教育委員会の承認が必要

### このプロダクトで解決されること
- 役員不在でも PTA 活動が回る
- 保護者の意見が AI で集約され、何が議題かが明確
- 操作ログ公開で透明性を担保
- AI による会費自動変更が「不可能な設計」で安心

### 典型的な利用シナリオ
- 初期導入時: 管理画面でアカウント設定・学校コード発行・保護者へ QR コード配布
- 日次: 管理画面で目安箱投稿の傾向確認
- 緊急時: 管理画面の「即時通知配信ボタン」で全保護者に連絡
- アンケート配信後: AI 結果分析の判断理由を確認
- イベント企画案生成時: 内容を確認し、必要なら学校側調整
- 月初: 月次レポート確認

### マッピング機能
F-05（ログイン・MFA）/ F-06 全機能 / F-01（投稿傾向確認）/ F-02（配信状況確認）/ F-08（企画案承認確認）/ F-09（管理者向け Coming Soon）/ NFR-MON-02（Slack 受信）

## P-04: 佐藤美香（学校の先生・PTA担当）

### 基本情報
| 項目 | 値 |
|---|---|
| 年齢 | 34歳 |
| 職業 | 公立小学校 担任教諭（5年生） |
| PCリテラシー | 中 |
| 主要デバイス | 学校PC + 個人iPhone |
| 役割 | PTA 会議の窓口役・保護者連絡 |

### モチベーション
- 担任業務に集中したい、PTA 関連で時間を取られたくない
- 保護者との連絡は適切に伝わってほしい
- 不要なメールは受け取りたくない

### 困りごと
- 保護者からの意見が直接来るとき対応が大変
- 緊急性の判断がつかず、すべて校長に上げてしまう
- アンケート意見主張トグル（Phase 2 機能）の負担懸念

### このプロダクトで解決されること
- 通常意見は AI が集約し、先生に直接来ない（MVP では先生エスカレーションは Phase 2）
- 重要事項のみ管理画面経由で共有される
- メール受信を選択可能（管理者設定で制御）

### 典型的な利用シナリオ（MVP範囲）
- 必要時のみ: 管理者からの転送メールを確認
- ほぼ操作なし（MVP では学校管理者画面の閲覧権限は校長のみと想定）

### マッピング機能
（MVP範囲では限定的）F-06（管理者からの情報受信）/ NFR-MON-02 補足対象

## P-05: 中村あかり（新入学保護者）

### 基本情報
| 項目 | 値 |
|---|---|
| 年齢 | 35歳 |
| 職業 | 会社員（時短勤務） |
| 子ども | 小学1年生（4月入学・1人目） |
| PCリテラシー | 中 |
| 主要デバイス | Android スマホ |
| PTA歴 | 初めて |

### モチベーション
- 入学したばかりで PTA がどんなものか分からない
- 最初から新しい仕組みに慣れたい
- 既存の慣習に縛られたくない

### 困りごと
- 入学式・懇談会でアプリ案内を受けても、インストール・登録手順が分からない
- 学校コードを入力する意味が不明
- 既存保護者の話題についていけない

### このプロダクトで解決されること
- QR コード → アプリインストール → 学校コード入力 → 登録 の導線が直感的
- 過去の月次レポートで学校・PTAの活動を把握できる
- 最初から「会議なし」が前提で違和感がない

### 典型的な利用シナリオ
- 入学式直後: QRコードからアプリインストール → メール+パスワード+学校コードで登録
- 1週目: アプリを開いて画面構成を確認、過去の月次レポートを読む
- 1〜3ヶ月目: 慣れてきたら目安箱に投稿、アンケート回答開始
- 不定期: Coming Soon ナビを見て「将来こんな機能が増えるんだ」と理解

### マッピング機能
F-05（初回登録の主対象）/ F-01〜F-04（基本機能の初回利用）/ F-08 / F-09（拡張性の理解）
