## F-01: デジタル目安箱

### US-F01-001: 目安箱への投稿（一般保護者）
**優先度**: Must
**ペルソナ**: P-01 一般保護者・P-02 意欲的保護者・P-05 新入学保護者

**As a** 一般保護者として、
**I want** 子どもの学校について気になることや意見をスマホアプリから簡単に投稿したい、
**so that** 学校に直接行ったり会議に出たりしなくても自分の意見を伝えられる。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given 保護者としてログイン済みである
When 「目安箱に投稿」ボタンをタップし、500文字以内のテキストを入力する
And 「匿名で投稿する」チェックボックスの選択を確定する
And 「投稿する」ボタンをタップする
Then 投稿は即座に DynamoDB に保存される（status=PENDING）
And 投稿一覧に自分の投稿が表示される
And 入力フォームが空にリセットされる

Given 保護者が500文字を超える投稿を送信しようとする
When 「投稿する」ボタンをタップする
Then エラーメッセージ「500文字以内で入力してください」が表示される
And 投稿は送信されない

Given 保護者が空文字を送信しようとする
When 「投稿する」ボタンをタップする
Then 「投稿する」ボタンが無効状態になっている（disabled）
```

**関連要件**: F-01.1, F-01.2 / SECURITY-05

### US-F01-002: 投稿の自動カテゴリ分類による絞り込み（一般保護者）
**優先度**: Must
**ペルソナ**: P-01 一般保護者

**As a** 一般保護者として、
**I want** 投稿一覧を「安全」「イベント」「連絡」「要望」「その他」のカテゴリで絞り込みたい、
**so that** 自分が関心のあるトピックの投稿だけを効率よく確認できる。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given 目安箱に複数の投稿が存在する
When 投稿一覧画面で「カテゴリ: 安全」を選択する
Then SAFETY カテゴリに分類された投稿のみが表示される

Given 新規投稿が直前にされた
When 一覧画面を更新する
Then その投稿が AI（Haiku）により自動分類されたカテゴリラベル付きで表示される
And 分類は SAFETY / EVENT / CONTACT / REQUEST / OTHER のいずれか1つである
```

**関連要件**: F-01.3, F-01.4 / AI-ARCH-04（Haiku 4.5 = 投稿分類）
**AI視点**: 保護者は「結果」のみを体験（カテゴリラベルが自動で付いている）

### US-F01-003: 投稿への支持表明（意欲的保護者）
**優先度**: Must
**ペルソナ**: P-02 意欲的保護者・P-01 一般保護者

**As a** 意欲的保護者として、
**I want** 他保護者の投稿に「支持」を1回まで表明したい、
**so that** 自分が賛同する意見を可視化し、組織で取り上げる優先順位を高められる。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given 自分以外の投稿が一覧に存在する
When その投稿の「支持」ボタンをタップする
Then 支持数が+1される
And 同じ投稿の「支持」ボタンが「支持済み」表示に変わる

Given 既に支持済みの投稿がある
When その投稿の「支持済み」ボタンをタップする
Then 何も起こらない（重複支持の防止）
And DynamoDB レベルでも重複登録が防止される
```

**関連要件**: F-01.5

### US-F01-004: 投稿対応状況の閲覧（一般保護者）
**優先度**: Should
**ペルソナ**: P-01 一般保護者・P-02 意欲的保護者

**As a** 保護者として、
**I want** 投稿が「未対応 / 検討中 / 実施済み」のどの状態にあるかを知りたい、
**so that** 自分や他人の意見が学校でどう扱われているかを透明に把握できる。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given 管理者が投稿の対応状況を更新済みである
When 保護者が投稿一覧を閲覧する
Then 各投稿に「未対応」「検討中」「実施済み」のステータスバッジが表示される

Given 投稿のステータスが管理者により「実施済み」に変更された
When 保護者がその投稿の詳細を表示する
Then 「実施済み」と表示される（時刻は MVP 範囲外でも可）
```

**関連要件**: F-01.6 / 公開操作ログ（NFR-COMP）

### US-F01-005: 匿名投稿のプライバシー保護
**優先度**: Must
**ペルソナ**: P-01 一般保護者

**As a** 保護者として、
**I want** 「匿名で投稿する」を選んだとき、自分のユーザーIDが他人や管理者から特定できないようにしたい、
**so that** デリケートな内容も気兼ねなく投稿できる。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given 保護者が「匿名で投稿する」を選択して投稿した
When 自分または他保護者または管理者が投稿一覧でその投稿を確認する
Then 投稿者欄には "ANONYMOUS" のみ表示される
And DynamoDB の userId 欄には実ユーザーIDが保存されていない（"ANONYMOUS" 文字列のみ）

Given 管理者がアプリログを確認する
When 該当投稿に関するログを検索する
Then ログにも実ユーザーIDが含まれていない（PII保護）
```

**関連要件**: F-01.2 / SECURITY-03（PII禁止）/ NFR-COMP-02

## F-02: AIアンケート自動生成 + 結果分析AI

### US-F02-001: AIによるアンケート配信判断（一般保護者の体験）
**優先度**: Must
**ペルソナ**: P-01 一般保護者

**As a** 一般保護者として、
**I want** アンケートが「必要なときだけ」配信されてほしい、
**so that** 通知疲れを起こさず、本当に意見が必要なときだけ回答すればよい。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given 直近の目安箱投稿数が閾値を超えている、または前回配信から十分な経過時間がある
When 毎日9時の EventBridge トリガーが発火する
Then 配信判断 AI（Sonnet）が「配信あり」と判定する
And 続けて質問生成 AI が5〜7問を生成する
And 全保護者にプッシュ通知が配信される

Given 直近の投稿数が少なく、配信不要と判断される条件である
When 毎日9時のトリガーが発火する
Then 配信判断 AI が「配信スキップ」と判定する
And 当日の配信は行われない
And CloudWatch Logs に「skip理由」が記録される
```

**関連要件**: F-02.1, F-02.3 / AI-ARCH 配信判断 Sub-Agent
**AI視点**: 保護者は「通知が来た / 来ない」という結果のみ体験

### US-F02-002: アンケート回答（一般保護者）
**優先度**: Must
**ペルソナ**: P-01 一般保護者

**As a** 一般保護者として、
**I want** プッシュ通知から1〜2タップでアンケート画面を開き、すべての設問に1分以内で回答したい、
**so that** 通勤電車などの隙間時間で参加できる。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given アンケート配信プッシュ通知を受信した
When 通知をタップする
Then アプリのアンケート画面が直接開く（深いリンク経由）
And アンケート設問が5〜7問表示される

Given アンケート画面で全設問に回答した
When 「送信」ボタンをタップする
Then 回答が DynamoDB に保存される
And 「ご回答ありがとうございました」のトーストが表示される
And アンケート結果のグラフ画面に遷移する
```

**関連要件**: F-02.5

### US-F02-003: アンケート結果のリアルタイム集計表示（意欲的保護者）
**優先度**: Must
**ペルソナ**: P-02 意欲的保護者・P-01 一般保護者

**As a** 意欲的保護者として、
**I want** 自分のアンケート回答後、他保護者の回答との集計結果をすぐに見たい、
**so that** 自分の意見が全体のなかでどの位置にあるかが分かる。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given 保護者がアンケートに回答送信した
When 結果画面が表示される
Then Recharts によるグラフ（バー or 円）で各設問の集計が表示される
And 自分の回答に「あなたの回答」マークが付く

Given 他保護者が新たに回答を送信した
When 自分が結果画面を再度開く
Then 集計値が更新されている（DynamoDB UpdateItem 経由のリアルタイム集計）
```

**関連要件**: F-02.6 / 注: MVPはAIによる集計ではなく単純集計（明確化4-3 B）

### US-F02-004: アンケート結果からの次アクション提案（学校管理者）
**優先度**: Must
**ペルソナ**: P-03 学校管理者

**As a** 学校管理者として、
**I want** アンケート期日後に AI が結果を分析し、「次に何をすべきか」を提案してほしい、
**so that** PTA 活動の次のステップが自動で見える化され、判断負荷が下がる。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given 配信済みアンケートが期日（例: 配信から7日後）を迎えた
When 毎日9時の EventBridge トリガーが発火する
Then 結果分析 AI（Sonnet）が回答を分析する
And 「次のアクション」を判定: イベント企画 / 目安箱新トピック / アクション不要 のいずれか

Given 「イベント企画」と判定された
When 結果分析が完了する
Then イベント企画 AI（Event Planning Sub-Agent）が連鎖起動される
And 管理者画面に「次アクション判定理由」が記録される

Given 「目安箱新トピック」と判定された
When 結果分析が完了する
Then 管理者画面に通知が表示される（保護者向け配信は MVP では行わない）
```

**関連要件**: F-02.7, F-02.8, F-02.9 / AI-ARCH Survey Result Analysis Sub-Agent
**AI視点**: 管理者は AI の判定結果と理由を体験

### US-F02-005: AIによる質問内容の品質期待
**優先度**: Should
**ペルソナ**: P-02 意欲的保護者

**When** AI が自動生成したアンケート質問を見たとき、
**I want to** 「ありきたりな質問」ではなく学校の状況に即した質問だと感じたい、
**so I can** AI に対する信頼を醸成し、回答意欲を維持できる。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given アンケート質問が AI（Sonnet）により生成された
When 保護者がアンケート画面を開く
Then 質問は5〜7問
And 質問は過去の目安箱投稿傾向 + 季節性 + 学校行事を反映している

Given プロンプトのゴールデンテストセットが整備されている
When CI が実行される
Then 期待される入出力ペアと現行プロンプトの出力が許容差分内であることが検証される
```

**関連要件**: F-02.2 / AI-PROMPT-02（ゴールデンテスト）

## F-03: プッシュ通知

### US-F03-001: AIアンケート配信通知の受信
**優先度**: Must
**ペルソナ**: P-01 一般保護者・P-05 新入学保護者

**As a** 保護者として、
**I want** AI アンケートが配信されたタイミングでスマホにプッシュ通知が届いてほしい、
**so that** アプリを開く習慣がなくても回答機会を逃さない。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given OS の通知許可をアプリに与えている
And 配信判断 AI が「配信あり」と判定した
When SNS → FCM/APNs 経由でプッシュ通知が送信される
Then 5分以内に保護者のスマホに通知が届く
And 通知タップでアンケート画面が直接開く

Given OS の通知許可を拒否している
When プッシュ通知が送信される
Then 通知は届かない（アプリ内では普通に確認可能）
```

**関連要件**: F-03.1, F-03.2 / NFR-PERF-02（5分以内配信）

---

### US-F03-002: 管理者からの即時通知配信（学校管理者）
**優先度**: Must
**ペルソナ**: P-03 学校管理者

**As a** 学校管理者として、
**I want** 緊急時（例: 学校行事の急な変更）に管理画面のボタンから即時通知を全保護者に送信したい、
**so that** EventBridge の9時を待たずに重要情報を伝達できる。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given 学校管理者として MFA ログイン済み
When 管理画面の「即時通知配信」フォームに件名と本文を入力し送信する
Then SNS → FCM/APNs 経由でその学校の全保護者に通知が配信される
And 配信完了画面に配信件数が表示される

Given 通知本文が長すぎる、または空欄
When 送信ボタンをタップする
Then バリデーションエラーが表示され、配信されない
```

**関連要件**: F-03.2（明確化2-3 A）, F-06.4 / SECURITY-05（入力バリデーション）

## F-04: 月次レポート自動生成

### US-F04-001: 月次レポートの自動配信（一般保護者）
**優先度**: Must
**ペルソナ**: P-01 一般保護者・P-02 意欲的保護者・P-05 新入学保護者

**As a** 一般保護者として、
**I want** 毎月初に前月の PTA 活動内容と予算消化を AI 生成のレポートで把握したい、
**so that** 自分が関わった意見やイベントが、どう活動に反映されたか確認できる。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given 月初（毎月1日 0時 JST）の EventBridge トリガーが発火した
When 月次レポート Sub-Agent（Sonnet）が起動する
Then 前月の活動内容（投稿数・アンケート結果・実施イベント）と予算データを要約する
And レポートが DynamoDB に保存される

Given 月次レポートが保存された
When 保護者がアプリの「月次レポート」画面を開く
Then 最新のレポートが日本語で表示される（外国籍向け英語版生成は MVP 範囲外）

Given 過去複数月のレポートが蓄積されている
When 保護者が画面を切り替える
Then 過去のレポートも閲覧できる
```

**関連要件**: F-04.1, F-04.2 / 注: MVPではプッシュ通知配信なし（明確化2-2 A）

---

### US-F04-002: 学校管理者によるレポート確認
**優先度**: Should
**ペルソナ**: P-03 学校管理者

**As a** 学校管理者として、
**I want** 配信前に当月のレポートを確認できるようにしたい、
**so that** 不適切な内容（誤情報・PII漏洩など）が公開される前にチェックできる。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given 月次レポートが生成された
When 学校管理者が管理画面の「月次レポート」セクションを開く
Then 当月分のレポート内容が表示される
And 保護者公開状態の表示がある（公開中 / 公開前）

Note: MVPでは保護者公開を遅延する機能（承認フロー）は持たない。
管理者は内容確認のみ可能、不適切と判断した場合は別途運用フローで削除依頼。
```

**関連要件**: F-04.2 / OQ-13（次アクション判定基準）に関連

## F-05: 保護者アカウント認証

### US-F05-001: 学校コードを使った初回登録（新入学保護者）
**優先度**: Must
**ペルソナ**: P-05 新入学保護者・P-01 一般保護者・P-02 意欲的保護者

**As a** 新入学保護者として、
**I want** 入学式で配布された QR コードからアプリをインストールし、メール+パスワード+学校コードで簡単に登録したい、
**so that** 学校が変わるたびに別のアカウントを作る複雑さなく、シンプルに参加できる。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given QR コードからアプリストアに遷移し、アプリをインストールした
When 起動後の「初回登録」画面でメール・パスワード・学校コードを入力する
And 「登録」ボタンをタップする
Then Cognito SignUp API が呼ばれる
And Custom Attribute として `schoolId` が設定される
And メール認証コードが SES 経由で送信される

Given 保護者がメール認証コードを入力する
When 「認証」ボタンをタップする
Then アカウントが有効化される
And ホーム画面（Topページ）に遷移する

Given 入力した学校コードが無効、または有効期限切れである
When 「登録」ボタンをタップする
Then 「学校コードが正しくありません」とエラーメッセージが表示される
And アカウントは作成されない
```

**関連要件**: F-05.1, F-05.2 / SECURITY-12（認証）

---

### US-F05-002: ログイン・トークン検証（一般保護者）
**優先度**: Must
**ペルソナ**: P-01 一般保護者・全保護者

**As a** 保護者として、
**I want** メールとパスワードでログインし、以降はトークンで自動認証されたい、
**so that** 毎回パスワードを入力せずアプリを使える。

**受け入れ基準（Given-When-Then）**:
```gherkin
Given 登録済み保護者である
When ログイン画面でメール・パスワードを入力し送信する
Then Cognito 認証が成功する
And アクセストークン・IDトークン・リフレッシュトークンを受け取る
And ホーム画面に遷移する

Given API リクエストにトークンを含める
When Lambda Authorizer がトークンを検証する
Then `schoolId` クレームが正しく抽出される
And API が schoolId 範囲外のデータにアクセスしないよう保証される（IDOR防止）

Given トークンの有効期限が切れている
When API リクエストを行う
Then 401 Unauthorized が返る
And アプリ側でリフレッシュトークンによる再認証が試みられる
```

**関連要件**: F-05.3 / SECURITY-08（IDOR防止）/ SECURITY-12

---

### US-F05-003: 退会とデータ匿名化（一般保護者）
**優先度**: Should
**ペルソナ**: P-01 一般保護者

**As a** 保護者として、
